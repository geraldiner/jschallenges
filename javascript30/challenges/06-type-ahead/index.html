Day 06

Ajax Type Ahead

Problem

For today's prompt, we created a search box that lists the cities that contains the text input, somewhat in the same way Google search allows for text completion. We are pulling from an external json file that lists cities and their populations.

I decided to follow along with this video and used the starting file this time because I was pressed for time. Usually I'd try to set up the HTML and CSS myself, but couldn't today.

Solution

First we set up a fetch for the endpoint of the data we were using, which is a json file. With the fetch we attached a .then to turn the raw data into readable json, and another .then to push the data to our cities array.

We then created a function called findMatches that takes in a wordToMatch, which is the text we will be matching to the items in the cities array. Using a regex with the wordToMatch, we use it in a filter to return all the matches in the cities array.

Finally we have a displayMatches function that shows the matched items as you're typing. Again, we use a regex to match the wordToMatch that was typed in, and use that to add a class on a span to show it as highlighted.
