Day 14: Reference vs. Copy

Day 14

Reference vs. Copy

Summary

This is another lesson where we’re not exactly building anything, but going over important distinctions between references and copies, especially in objects and arrays.

With strings, numbers, and booleans, you can copy a variable to a new variable, just by assigning it from one to the other.

var name = "geraldine"
var name2 = name;

And if you console.log(name, name2), you will get

geraldine geraldine

And if you wanted to update one, without updating the other

name2 = "gigi"

And run console.log(name, name2) again, you will get

geraldine gigi

However, this does not work with arrays and objects. This:

var arr1 = [0, 1, 2, 3]
var arr2 = arr1

does not work in the same way that it does with strings in that it is only making a reference to arr1, which we call arr2.

So, while console.log(arr1, arr2) will give us the same result, updating arr2 will also update arr1.

Output for console.log(arr1, arr2):

[0, 1, 2, 3] // arr1
[0, 1, 2, 3] // arr2

Changing the first item in arr2 will also change the first item in arr1.

arr2[0] = 9

Output for console.log(arr1, arr2):

[9, 1, 2, 3] // arr1
[9, 1, 2, 3] // arr2

To have two separate arrays, you have to actually make a copy of it, either by using slice, concat, spread (…), or Array.from.

With objects, you use assign, adding in all the properties you want to overwrite. However, you still won’t be able to change properties afterwards. So you’d need to know beforehand everything you want to change and use it in assign. He also mentioned a "poor man’s version" where you stringify the object and then turn it back to JSON, and then change it as you wish.
