Day 08

HTML Canvas

Problem

Today is creating a drawing canvas in HTML, which is pretty cool. I love colors so I am easily excited about this one. I don’t know a lot about using the canvas in HTML, so I will be following along with the video. We want to start drawing once the user clicks the mouse. The color starts as red, but starts changing colors through the rainbow until it reaches red again, and it starts all over again. It also starts as a small brush size and increases in size until it reaches a certain size, and then it will start decreasing.

Solution

The only thing we needed on the HTML was the canvas tag set to a certain width and height. In JavaScript, we grabbed the canvas and set a context to draw on. (I had to look up what this all meant because I’d never heard of it before.) We have variables to help keep track of the mouse’s movements while it’s on the page. However, with the event listeners, it only draws when the mouse is being clicked.

In our draw function, we use the mouse’s offsetX and offsetY values to create a path, from where it was to where it’s going. If the mouse is moving really fast, then that means the function is being called instantly one after the other. The colors come from a hue variable that increments with each call, as does the line width. But there is an if statement to check whether it’s gone to a specific size, and when it does reach that size, it will start getting smaller again.

Lessons Learned
