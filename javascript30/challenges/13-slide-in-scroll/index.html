Day 13

Slide In On Scroll

Problem

Today we implemented the ‘slide in on scroll’ animation for images to slide in from the sides as you scroll down the page. I’ve seen this in Materialize, which they call ScrollFire. I have to agree with Wes and say that I’m not a fan of it either. It does seem cool, but from the UX perspective, I think content should already just be there and readily accessible. I think it also adds a bit of noise as you’re scrolling.

But I digress.

Solution

Wes provided us with a debounce function that takes in a function and only calls it after a certain amount of time. This is for the event listener on ‘scroll’ so we don’t have a million calls to it as the user is scrolling down the page. Instead, it only gets called after 20 seconds here.

After getting all the images from the document, we use a forEach loop to check where the height of the image is and calculate where the halfway point of the image is. We want the image to only start sliding in when bottom of the page has reached its halfway point. We add the .active class, which allows it to slide in. The same thing is applied when you scroll back up and the top of the page reaches the halfway point of the image. When the image is offscreen, then the .active class is removed and it is hidden again.

Lessons Learned

To be honest, the hardest part of this challenge was the math. And I feel so ashamed to admit that because I used to pride myself on my math skills in high school (the good old days). But also honestly, I think it’s just the variables I need to get used to using and knowing what they are. For the most part they’re descriptive, like height and offsetTop, and hearing Wes talk about them gave me a better understanding of what they are.

Per usual, I just need more practice!
